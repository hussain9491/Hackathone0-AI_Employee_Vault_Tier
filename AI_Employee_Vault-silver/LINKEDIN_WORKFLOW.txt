1. OVERVIEW
================================================================================

The LinkedIn Automation system has TWO main components:

┌─────────────────────────────────────────────────────────────────────────────┐
│  COMPONENT 1: LINKEDIN WATCHER (Input)                                      │
│  Purpose: Monitor LinkedIn for new activity                                 │
│  - Watches for connection requests                                          │
│  - Watches for new messages                                                 │
│  - Creates action files in Needs_Action/ folder                             │
│  - Runs continuously in background                                          │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  COMPONENT 2: LINKEDIN POSTER (Output)                                      │
│  Purpose: Create posts on LinkedIn                                          │
│  - Posts text content                                                       │
│  - Posts with images                                                        │
│  - Supports manual review before posting                                    │
│  - Can be automated with Qwen                                               │
└─────────────────────────────────────────────────────────────────────────────┘

IMPORTANT: The WATCHER does NOT create posts automatically.
           It only MONITORS for incoming activity.

2. HOW LINKEDIN WATCHER WORKS
================================================================================

STEP-BY-STEP PROCESS:
---------------------

Step 1: Start the Watcher
-------------------------
Command: python linkedin_watcher.py --vault-path ../AI_Employee_Vault

The watcher starts a continuous loop that runs every 5 minutes (default).


Step 2: Browser Launches (Headless)
------------------------------------
- Opens Chromium browser using Playwright
- Loads saved LinkedIn session from:
  AI_Employee_Vault/.obsidian/linkedin_session/
- Navigates to LinkedIn.com


Step 3: Check Connection Requests
----------------------------------
- Goes to: https://www.linkedin.com/mynetwork/
- Looks for elements with class: .invitation-card
- Extracts:
  * Person's name
  * Person's headline (job title, company)
- Checks if already processed (avoids duplicates)


Step 4: Check Messages
-----------------------
- Goes to: https://www.linkedin.com/messaging/
- Looks for conversations with unread indicator
- Extracts:
  * Sender name
  * Message preview/snippet
- Checks for LEAD_KEYWORDS like:
  * "interested", "pricing", "quote"
  * "demo", "service", "product"
  * "collaboration", "partnership"
  * "opportunity", "hire", "consulting"


Step 5: Create Action Files
----------------------------
For each new connection request:
  Creates: Needs_Action/LINKEDIN_CONNECT_Name_YYYYMMDD.md

For each new message:
  Creates: Needs_Action/LINKEDIN_MSG_Name_YYYYMMDD.md

Action file contains:
  - Frontmatter with metadata (type, priority, status)
  - Original message preview
  - Suggested actions checklist
  - Draft response template


Step 6: Track Processed Items
------------------------------
- Saves processed IDs to:
  Logs/linkedin_connections.txt
  Logs/linkedin_messages.txt
- Prevents duplicate action files


Step 7: Wait and Repeat
------------------------
- Waits 5 minutes (configurable)
- Repeats from Step 2

7. QWEN INTEGRATION - AUTOMATED WORKFLOW
================================================================================

WORKFLOW OVERVIEW:
------------------

1. Watcher detects new message/connection
2. Creates action file in Needs_Action/
3. Qwen reads action file
4. Qwen drafts response
5. Human approves (moves to Approved/)
6. Automated poster publishes


STEP-BY-STEP QWEN WORKFLOW:
----------------------------

Step 1: Qwen Checks for New Items
---------------------------------
Command: Qwen reads Needs_Action/ folder

Qwen looks for files matching:
- LINKEDIN_CONNECT_*.md (connection requests)
- LINKEDIN_MSG_*.md (messages)


Step 2: Qwen Processes Each File
---------------------------------
For each file, Qwen:
1. Reads the metadata (name, headline, message)
2. Understands the context
3. Drafts appropriate response
4. Updates the file with draft
5. Changes status to: draft_ready


Example Qwen Action:
--------------------
File: Needs_Action/LINKEDIN_CONNECT_John_Doe_20260315.md

Qwen adds draft:

## Welcome Message Draft

Hi John, thanks for connecting! I noticed you're CEO at Tech Corp.
I'd love to learn more about your work in the tech industry.
Best regards, Hussain


Step 3: Human Review
--------------------
User moves file:
Needs_Action/ → Pending_Approval/

Reviews draft, edits if needed


Step 4: Automated Posting (NEW!)
--------------------------------
The auto-poster script monitors the Approved/ folder:

python linkedin_auto_post.py --vault-path ../AI_Employee_Vault

- Finds approved files (LINKEDIN_POST_*.md)
- Extracts post content
- Posts to LinkedIn automatically
- Moves file to Done/ when complete
- Logs the result

================================================================================
8. TROUBLESHOOTING
================================================================================

PROBLEM: Watcher says "Not logged in"
--------------------------------------
Cause: Session expired

Solution:
1. Re-login: python linkedin_login.py
2. Or use visible mode once: python linkedin_poster.py --login --visible


PROBLEM: No action files created
--------------------------------
Causes:
- No new activity on LinkedIn
- Watcher not running
- Session invalid

Solutions:
1. Check if watcher is running
2. Verify session: python verify_linkedin.py
3. Check logs: Logs/linkedin_watcher_YYYYMMDD.log


PROBLEM: Posts not appearing on LinkedIn
-----------------------------------------
Causes:
- Post button not clicked (in visible mode)
- Browser closed too soon
- LinkedIn flagged account

Solutions:
1. Always use --visible flag for posting
2. Wait for manual confirmation
3. Don't post too frequently


PROBLEM: Browser opens but shows login page every time
-------------------------------------------------------
Cause: Session not being saved properly

Solution:
1. After login, DON'T close browser immediately
2. Scroll through feed for 10-15 seconds
3. Then close browser normally
4. This helps LinkedIn save the session cookies
